ğŸ“– Overview
This project is an end-to-end Machine Learning application designed to predict customer churn for a telecommunications company. It helps businesses identify customers who are likely to cancel their subscriptions and provides actionable insights through data analysis and customer segmentation.

The solution includes:
A Jupyter Notebook for data exploration, preprocessing, and model training.
A Streamlit Web Application for real-time predictions, batch processing, and visual analytics.

âœ¨ Key Features
Real-time Prediction: Enter individual customer details (Age, Tenure, Contract Type, etc.) to get an instant Churn/No-Churn prediction.
Batch Prediction: Upload a CSV or Excel file containing multiple customer records to generate predictions for the whole dataset.
Interactive Dashboard: Visualizes churn distribution, correlation heatmaps, and relationships between tenure and monthly charges.
Customer Segmentation: Uses K-Means Clustering to group customers into "Loyal," "At-Risk," and "Moderate Risk" categories to suggest retention strategies.
Actionable Insights: Provides automated suggestions based on data patterns (e.g., offering discounts to at-risk users).

ğŸ§  Model Workflow
The machine learning pipeline implemented in Churn_nb.ipynb follows these steps:

Data Loading: Reads customer data (Customer ID, Gender, Tenure, Charges, etc.).

Preprocessing:
Encodes categorical variables (Gender, ContractType, InternetService, TechSupport).
Scales numerical features using StandardScaler.
Model Selection: Tested multiple algorithms:
Logistic Regression (~89.5% accuracy)
K-Neighbors Classifier (~94% accuracy)
Support Vector Classifier (SVC) (~96% accuracy)
Decision Tree Classifier (~100% accuracy on test set - Selected Model)
Hyperparameter Tuning: Used GridSearchCV to optimize the Decision Tree.
